# 📊 Analytics Setup Guide

Panduan lengkap untuk setup dan populate data analytics di Azure Cosmos DB.

---

## 🎯 Overview

Script `seed-analytics-data.ts` akan generate:
- ✅ **Analytics Daily** - Data analitik harian untuk 7 hari terakhir
- ✅ **Traffic Data** - Data lalu lintas real-time (10 records per intersection per day)
- ✅ **Events** - Kejadian dan alerts (2-3 per intersection per day)

Total data yang akan dibuat:
- Analytics: ~28 records (7 days × 4 intersections)
- Traffic: ~280 records (7 days × 4 intersections × 10 records)
- Events: ~56 records (7 days × 4 intersections × 2-3 events)

---

## 🚀 Quick Start

### 1. Pastikan Database Sudah Setup

```bash
# Check apakah intersections sudah ada
npm run db:check

# Jika belum, setup dulu
npm run db:setup
npm run db:seed
```

### 2. Run Analytics Seeding

```bash
# Seed analytics data
npm run db:seed:analytics
```

### 3. Verifikasi Data

```bash
# Start dev server
npm run dev

# Buka halaman Analist
# http://localhost:3000/Analist

# Atau test API
curl http://localhost:3000/api/analytics/daily
curl http://localhost:3000/api/traffic/realtime
curl http://localhost:3000/api/events
```

## 📊 Data yang Dihasilkan

### 1. Analytics Daily

**Structure:**
```typescript
{
  id: "int_001_2026-04-20",
  intersectionId: "int_001",
  date: "2026-04-20",
  summary: {
    totalVehicles: 5420,
    averageSpeed: 32.5,
    averageCongestionIndex: 65.2,
    averageWaitTime: 45.3,
    peakHour: "08:00",
    peakVehicleCount: 680
  },
  hourly: [
    {
      hour: 0,
      vehicleCount: 120,
      averageSpeed: 45,
      congestionLevel: "low",
      congestionIndex: 15
    },
    // ... 23 hours total
  ],
  efficiency: {
    autoModeTime: 1320,        // 22 jam dalam menit
    manualModeTime: 120,       // 2 jam dalam menit
    autoModeEfficiency: 92.5,
    manualModeEfficiency: 68.3
  },
  events: {
    total: 5,
    bySeverity: {
      low: 2,
      medium: 2,
      high: 1,
      critical: 0
    },
    byType: {
      congestion: 3,
      accident: 0,
      sensor_error: 1,
      other: 1
    }
  }
}
```

**Pattern Hourly Data:**
- **00:00-04:00** - Rendah (20-50 kendaraan)
- **05:00-06:00** - Mulai meningkat (80-200 kendaraan)
- **07:00-09:00** - **Peak pagi** (350-450 kendaraan)
- **10:00-15:00** - Moderat (250-320 kendaraan)
- **16:00-18:00** - **Peak sore** (380-480 kendaraan)
- **19:00-23:00** - Menurun (60-200 kendaraan)

### 2. Traffic Data

**Structure:**
```typescript
{
  id: "ESP32_001-north-1713600000000",
  deviceId: "ESP32_001",
  lane: "north",
  vehicleCount: 85,
  speed: 35,
  density: 56,
  status: "congested",
  timestamp: "2026-04-20T08:30:00.000Z",
  _ts: 1713600000
}
```

**Status Logic:**
- `vehicleCount < 50` → status: "normal"
- `vehicleCount 50-100` → status: "congested"
- `vehicleCount > 100` → status: "critical"

**Speed Logic:**
- `vehicleCount > 100` → speed: 15-30 km/h (macet)
- `vehicleCount ≤ 100` → speed: 30-60 km/h (lancar)

### 3. Events

**Structure:**
```typescript
{
  id: "evt_1713600000_1234",
  intersectionId: "int_001",
  type: "congestion",
  priority: "high",
  status: "open",
  title: "Kemacetan Parah Terdeteksi",
  description: "Event terdeteksi di Simpang Empat Diponegoro. Memerlukan perhatian high.",
  timestamp: "2026-04-20T08:30:00.000Z",
  metadata: {
    intersectionName: "Simpang Empat Diponegoro",
    autoGenerated: true
  }
}
```

**Event Types:**
- `congestion` - Kemacetan terdeteksi
- `sensor_error` - Error pada sensor
- `manual_override` - Mode manual diaktifkan
- `system_alert` - Alert sistem

**Priority Levels:**
- `low` - Informasi biasa
- `medium` - Perlu perhatian
- `high` - Urgent
- `critical` - Sangat urgent

## 🎨 Customization

### Ubah Date Range

Edit `scripts/seed-analytics-data.ts`:

```typescript
// Generate untuk 30 hari terakhir (default: 7 hari)
for (let i = 0; i < 30; i++) {
  const date = new Date(today);
  date.setDate(date.getDate() - i);
  // ...
}
```

### Ubah Traffic Pattern

Edit function `generateHourlyData()`:

```typescript
const patterns = [
  { hour: 0, base: 50, variance: 20 },
  { hour: 1, base: 30, variance: 15 },
  // Ubah base dan variance sesuai kebutuhan
  // base: rata-rata kendaraan
  // variance: range variasi
];
```

### Ubah Event Frequency

Edit function `generateEvent()`:

```typescript
// Generate 5-8 events per day (default: 2-3)
const eventCount = randomInt(5, 8);
for (let j = 0; j < eventCount; j++) {
  const event = generateEvent(intersection.id, intersection.name);
  eventsData.push(event);
}
```

## 📈 Verifikasi Data di Azure

### 1. Buka Data Explorer

1. Login ke [Azure Portal](https://portal.azure.com)
2. Navigate ke Cosmos DB: `traffic-cosmos-slam`
3. Klik **Data Explorer**

### 2. Query Analytics Data

```sql
-- Lihat semua analytics
SELECT * FROM c ORDER BY c.date DESC

-- Count analytics per intersection
SELECT c.intersectionId, COUNT(1) as count
FROM c
GROUP BY c.intersectionId

-- Lihat analytics hari ini
SELECT * FROM c WHERE c.date = "2026-04-20"

-- Lihat peak hours
SELECT c.intersectionId, c.date, c.summary.peakHour, c.summary.peakVehicleCount
FROM c
ORDER BY c.summary.peakVehicleCount DESC
```

### 3. Query Traffic Data

```sql
-- Lihat traffic data terbaru
SELECT TOP 100 * FROM c ORDER BY c._ts DESC

-- Count per device
SELECT c.deviceId, COUNT(1) as count
FROM c
GROUP BY c.deviceId

-- Lihat traffic dengan status critical
SELECT * FROM c WHERE c.status = "critical"
```

### 4. Query Events

```sql
-- Lihat events yang open
SELECT * FROM c WHERE c.status = "open"

-- Count events by type
SELECT c.type, COUNT(1) as count
FROM c
GROUP BY c.type

-- Lihat critical events
SELECT * FROM c WHERE c.priority = "critical"
```

## 🐛 Troubleshooting

### Issue: "No intersections found"

**Solution:**
```bash
# Seed intersections dulu
npm run db:seed

# Lalu seed analytics
npm run db:seed:analytics
```

### Issue: "Duplicate key error"

**Solution:**
```bash
# Cleanup dulu
npm run db:cleanup

# Atau hapus manual di Azure Data Explorer
# DELETE FROM c WHERE c.date >= "2026-04-13"

# Lalu seed ulang
npm run db:seed:analytics
```

### Issue: "Connection timeout"

**Solution:**
```bash
# Check environment variables
cat .env.local

# Verify credentials di Azure Portal
# Restart dev server
npm run dev
```

### Issue: "Data tidak muncul di frontend"

**Solution:**
1. Check API response:
   ```bash
   curl http://localhost:3000/api/analytics/daily
   ```

2. Check browser console untuk errors

3. Verify data di Azure Data Explorer

4. Clear browser cache (Ctrl+Shift+R)

## 📊 Data Visualization

### Halaman Analist

Setelah seeding, buka http://localhost:3000/Analist untuk melihat:

1. **Analisis Kepadatan Kendaraan**
   - Bar chart mingguan per jalur (Utara, Timur, Barat, Selatan)
   - Data dari `analytics_daily.hourly`

2. **Laporan Kepadatan Per Jam**
   - Heatmap 24 jam
   - Color-coded berdasarkan intensity
   - Data dari `analytics_daily.hourly`

3. **Performa Sensor IoT**
   - Akurasi deteksi
   - Perangkat aktif
   - Data dari `intersections` dan `device_status`

4. **Indeks Kemacetan**
   - Circular progress indicator
   - Real-time congestion level
   - Data dari `analytics_daily.summary.averageCongestionIndex`

5. **Peringatan Kritis**
   - List events dengan priority high/critical
   - Data dari `events` collection
